# DevTrail

**A developer's trail on every work item.** / **La trace de chaque développement, carte par carte.**

DevTrail adds a history section to your Azure Boards work item form, right next to Acceptance Criteria. Developers log the full story of a task — blockers, solutions, progress, notes — as styled markdown cards saved into the work item's native Discussion.

## Features

- ✍️ **Markdown composer** with live preview
- 🏷️ **Four card types**: ⚠ Blocker (red), 🔧 Solution (blue), ✔ Progress (green, default), ℹ Info (gray)
- 🖼️ **Inline images** in comments
- 🔒 **Assignee-only posting** — only the person the item is assigned to writes the trail
- 🌍 **French & English** UI, auto-detected
- 💾 **No lock-in** — everything is stored as native work item comments

## Getting started

1. Install the extension in your organization.
2. Open any work item — the **DevTrail** group appears on the form.
3. Pick a type, write in markdown, post. Your trail begins.
